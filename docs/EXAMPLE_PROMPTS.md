## Prompt Template

```text
Use $skill-name to [build/restyle/create] a [product surface] for [audience].
Include [key sections].
Preserve [existing constraints].
Support [theme requirements].
Show [important UI states].
Keep the result [standalone/production-ready/mobile-aware].
```

## Useful Constraints

- Ask for both light and dark themes when the skill supports dual-theme output.
- Name the target product surface clearly: dashboard, admin panel, AI workspace, settings screen, internal tool, or productivity app.
- Mention required states: loading, empty, success, warning, error, disabled, hover, and focus.
- Ask Codex to keep examples standalone when the result should be easy to inspect.
- Include accessibility expectations when the UI has forms, dense data, or complex controls.
