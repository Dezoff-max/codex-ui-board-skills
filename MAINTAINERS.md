# Maintainers

This project is maintained by:

| Maintainer | Role | Scope |
| --- | --- | --- |
| [@Dezoff-max](https://github.com/Dezoff-max) | Owner and primary maintainer | Repository direction, releases, Codex skill structure, review, and security triage |

## Maintainer Responsibilities

- Review pull requests for skill structure, design-token consistency, screenshots, and documentation quality.
- Triage issues with labels such as `tooling`, `examples`, `new-skill`, `quality`, and `documentation`.
- Keep `npm run validate` and the `Codex Skill Lint` workflow passing on `main`.
- Publish releases with changelog notes, source archives, and gallery links.
- Review security reports privately through the process in [SECURITY.md](./SECURITY.md).

## Review Expectations

Maintainers should prefer small, focused pull requests. New skills should include:

- a standalone preview
- screenshots
- design tokens
- CSS and Tailwind preset references
- Codex skill metadata
- usage documentation
- passing validation
